# 2026-03-11 Global-Install Stabilization

This note archives the milestone work that completed the transition to MDT's
global-first install model and the immediate documentation stabilization that
followed.

## Completed Install-Model Changes

- normal MDT installs now target the tool's user/global config root
- `--global` remains accepted as a compatibility alias/no-op
- `--project-dir` is retired
- MDT-owned state now lives under `~/.{tool}/mdt/`
- repo-local exceptions are handled as explicit bridge flows instead of full
  project-local installs

## Completed Runtime/Tooling Changes

- Cursor lifecycle hooks now use native Cursor payloads for session summaries
  and evaluation instead of depending on Claude `transcript_path`
- the Cursor adapter now guarantees `MDT_ROOT` for delegated subprocesses
- Cursor cost tracking now records usage when payload data is present and logs
  an explicit fallback when it is not
- focused Cursor lifecycle tests now cover the native session-end/stop path
- Codex installs are package-driven and materialize package-selected skills into
  `~/.codex/skills/`
- Codex `continuous-learning` now uses an explicit manual workflow with
  project-scoped state under `~/.codex/mdt/homunculus/`

## Completed Doc/Taxonomy Cleanup

- active install-contract docs were updated to match the global-first model
- manual verification pages were updated to point at installed global paths
- `NEXT-STEPS.md` was trimmed back to active roadmap items
- `BACKLOG.md` was trimmed back to deferred open work
- `docs/session-handoff.md` was converted into a historical stub
- the completed backlog items below were archived out of the live backlog:
  - replacing hardcoded generic `~/.claude/` references with tool-aware paths
  - establishing core local-first setup/workflow verification
  - splitting continuous-learning into manual and hook-driven skills
