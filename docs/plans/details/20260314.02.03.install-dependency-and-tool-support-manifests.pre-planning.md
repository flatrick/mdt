## Risks to address

### R1. Scope creep — generated surfaces ✓

**Mitigation:** "Generated install surfaces are out of scope for this plan" added explicitly to the Decisions block. Template directory layout remains hand-maintained. Follow-up plan gated on resolver stability.

- [x] Mitigation reflected in [main plan](20260314.02.03.install-dependency-and-tool-support-manifests.md) decisions.

---

### R2. Breaking existing install behavior ✓

**Mitigation:** "Equivalent closure" (same files per package+tool) defined in migration workstream. Snapshot tests must verify equivalence before flipping the default. `--new-resolver` flag keeps old path live until migration is complete.

- [x] Mitigation reflected in [main plan](20260314.02.03.install-dependency-and-tool-support-manifests.md) migration workstream.

---

### R3. Doc/source-of-truth drift ✓

**Mitigation:** `scripts/ci/validate-support-maps.js` added to the validation workstream. Exits 0 with warning when `metadata/tools/` does not yet exist; hard-fails once any file is present but diverges from `docs/tools/capability-matrix.md`. Wired to `run-test-pipeline.js` in the same commit that creates the first tool file.

- [x] Mitigation reflected in [main plan](20260314.02.03.install-dependency-and-tool-support-manifests.md) validation workstream.

---

### R4. Blocking ai-learning and observer flows ✓

[AI-Learning Self-Contained](20260314.12.04.ai-learning-self-contained.md) is **halted** on this plan. Slip here directly delays the ai-learning vertical slice and any flows that depend on the optional Codex observer.

**Mitigation:** First vertical slice workstream tightly scoped to `ai-learning` rename + sidecars + optional Codex observer. Four explicit, testable unblock criteria added to the ai-learning plan's "Workstream: unblock criteria" section: (1) schema validators in CI with a real sidecar file, (2) optional-capability path in resolver tested, (3) first tool support map validated in CI, (4) first vertical slice (rename + sidecars) landed.

- [x] Unblock criteria added to the [ai-learning plan](20260314.12.04.ai-learning-self-contained.md).

---

### R5. Compose redesign dependency (partial)

The [Composable MDT Redesign](20260314.12.04.composable-mdt-redesign.md) will consume this plan's schema and resolver. If the contract changes later (new capability types, different sidecar shape), compose work may need rework.

**Mitigation (partial):** Schema `version` field (`v0.0.1`) defined in the discovery plan Q1. A stability contract (e.g. additive-only changes for N releases) has not yet been written.

- [~] Schema version field done. Stability contract TBD — document in a shared design doc or add an "Additive-only" policy note to `docs/package-manifest-schema.md` when it is updated post-implementation.

---

### R6. Optional dependency UX and exit codes ✓

**Mitigation:** Exit code contract added to the resolver workstream: exit 0 when all required capabilities satisfied (optional-skipped fine); non-zero when any required unsatisfied or required+experimental without `--experimental`. Distinct user message for experimental vs truly absent. Dry-run: human-readable by default, `--format=json` for structured output.

- [x] Mitigation reflected in [main plan](20260314.02.03.install-dependency-and-tool-support-manifests.md) resolver workstream.

---

### R7. Cycle detection and extends ✓

Package `extends` cycle detection already exists in `validateExtendsGraph` (`validate-install-packages.js`). The new sidecar `requires` graph is a cross-file concern that can only be detected during graph assembly — it belongs in the resolver, not a standalone schema validator (schema validators check individual file shape only).

**Resolution:** Cycle detection bullet moved from validation workstream to resolver workstream. Graph scope (three edge types: `extends`, package→artifact, sidecar `requires`) and error format (full path to stderr, all cycles reported before exit, non-zero) are now specified in the resolver workstream.

- [x] Mitigation reflected in [main plan](20260314.02.03.install-dependency-and-tool-support-manifests.md) resolver workstream.

---

### R8. Enforcement of "add-ons = metadata/docs/config only" ✓

**Mitigation:** E2E check added to the validation workstream: install to a tmp folder, assert only allowlisted file types/directories appear in add-on dirs. Allowlist: `skill.meta.json`, `SKILL.md`, `STYLE_PRESETS.md`, `config.json`, `agents/`, `rules/`.

- [x] Mitigation reflected in [main plan](20260314.02.03.install-dependency-and-tool-support-manifests.md) validation workstream.
