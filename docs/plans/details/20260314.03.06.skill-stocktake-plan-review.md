**Add to the consolidation plan:**

- **Intent:** "The stocktakes ran against Claude's installed configs; the goal is to improve **what MDT installs**. Implement these changes in the **repo** (commands/, skills/, claude-template/, installer, etc.) so that future installs deliver the consolidated surface. Updating only ~/.claude or ~/.cursor would be overwritten on the next install."
- **Scope:** "The audits targeted **Claude** (`~/.claude/`) and **Cursor** (`~/.cursor/skills/`). Apply the same merges/improvements to the repo's Claude-related surfaces (and Cursor/Codex templates where equivalent assets exist)."
- **Order:** "Execute **M1 before M2** (M2 retires docs-health, which references documentation-steward retired in M1). M3 and M4 are independent and can run in any order."
- **Where to change:** For M1 and M3, apply the same principle: remove or merge in the **repo** (e.g. remove `documentation-steward` from skills/templates if present; remove `learn.md` from commands/ and templates, add the /learn-eval note to repo `ai-learning.md`) so that what the installer deploys is already consolidated.
