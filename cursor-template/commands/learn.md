# Learn Command

Use this command to reflect on the current session and extract reusable knowledge.

## What To Extract

Look for:

1. Error-resolution patterns
2. Debugging techniques that worked well
3. Reliable workarounds
4. Project-specific workflow conventions
5. Candidate skill, rule, or checklist updates

## Output Goal

Summarize:
- what happened
- what was learned
- what is reusable
- whether the insight should become a skill, instinct, rule, or doc update

## Preferred Output Structure

```md
# Session Learning Summary

## What Was Accomplished
- ...

## Reusable Patterns
- ...

## Candidate Follow-Ups
- skill update:
- rule update:
- docs update:
```

## Required Behavior

- Focus on non-trivial lessons.
- Do not extract one-off noise.
- Prefer patterns that will save time in future sessions.
- Ask for confirmation before writing durable learned artifacts if the user has not requested that already.
